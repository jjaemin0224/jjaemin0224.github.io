[개념] 프로그래밍 패러다임

프로그래밍 패러다임

프로그래밍 패러다임(Programming Paradigm)은 프로그램을 수행하는 방식과 접근 방법을 기술하는 개념입니다. 언어와는 별개로, 프로갬의 구조와 작성 방법을 결정하는 원리와 철학을 의미합니다. 다양한 프로그래밍 패러다임은 다른 문제 해결 방식과 코드의 구조를 제공하며, 특정한 상황이나 요구사항에 적합한 패러다임을 선택할 수 있습니다.

1. 절차 지향 프로그래밍(Procedural Programming)

  • 프로그램을 순차적인 단계 또는 절차로 구성합니다.
  • 상태와 동작을 분리하여 프로시저(서브루틴 또는 함수)를 사용하여 문제를 해결합니다.
  • 코드의 재사용을 위해 모듈화를 강조합니다.
  • C, Pascal, Fortran 등이 절차 지향 프로그래밍 언어의 예시입니다.
#include <stdio.h>

// 두 정수의 합을 계산하는 함수
int sum(int a, int b) {
    return a + b;
}

int main() {
    int num1 = 10;
    int num2 = 20;
    int result = sum(num1, num2);
    printf("두 수의 합: %d\n", result);
    return 0;
}

2. 객체 지향 프로그래밍(Object-Oriented Programming : OOP)

  • 프로그램을 상호작용하는 객체들의 집합으로 구성합니다.
  • 객체는 데이터와 그 데이터를 처리하는 메서드를 포함합니다.
  • 캡슐화, 상속, 다형성 등의 개념을 활용하여 코드의 재사용성과 유연성을 높입니다.
  • Java, C++, Python 등이 객체지향 프로그래밍 언어의 예시입니다.
 public class Circle {
    private double radius;
    
    public Circle(double radius) {
        this.radius = radius;
    }
    
    public double getArea() {
        return Math.PI * radius * radius;
    }
}

public class Main {
    public static void main(String[] args) {
        Circle circle = new Circle(5.0);
        double area = circle.getArea();
        System.out.println("원의 넓이: " + area);
    }
}

3. 함수형 프로그래밍(Functional Programming)

  • 프로그램을 수학적인 함수의 평가로 구성합니다.
  • 상태 변경을 피하고 불변성을 강조합니다.
  • 함수 조합과 재귀를 사용하여 문제를 해결합니다.
  • Haskell, Lisp, Clojure 등이 함수형 프로그래밍 언어의 예시입니다.
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

def main():
    num1 = 5
    num2 = 10
    result = multiply(add(num1, num2), num2)
    print("결과:", result)

if __name__ == "__main__":
    main()

4. 논리형 프로그래밍(Logic Programming)

  • 프로그램을 명시적인 사실과 규칙의 집합으로 구성합니다.
  • 문제를 해결하기 위해 명제 논리와 추론을 사용합니다.
  • Prolog가 논리형 프로그래밍 언어의 대표적인 예시입니다.
% 사람인지 확인하는 규칙
person(john).
person(mary).

% 부모-자식 관계를 정의하는 규칙
parent(john, mary).
parent(john, jim).

% 조상-자손 관계를 정의하는 규칙
ancestor(X, Y) :- parent(X, Y).
ancestor(X, Y) :- parent(X, Z), ancestor(Z, Y).

% 실행 예시
?- ancestor(john, jim).

5. 이벤트 주도 프로그래밍(Event-driven Programming)

  • 프로그램의 제어 흐름이 이벤트(사용자 입력, 센서 신호 등)에 따라 결정됩니다.
  • 이벤트 발생 시 적절한 이벤트 핸들러를 호출하여 처리합니다.
  • GUI 애플리케이션과 상호작용하는 데 효과적입니다.
  • JavaScript, Visual Basic 등이 이벤트 주도 프로그래밍의 예시입니다.
const button = document.querySelector("#myButton");
button.addEventListener("click", function() {
    console.log("버튼이 클릭되었습니다!");
});

이외에도 많은 프로그래밍 패러다임이 존재하며, 각각의 패러다임은 특정한 상황에서 뛰어난 효과를 발휘할 수 있습니다. 패러다임은 프로그래머에게 코드 구조와 작성 방법에 대한 지침을 제공하고, 코드의 유지보수성, 재사용성, 가독성 등을 개선할 수 있습니다. 프로그래머는 문제의 복잡성과 요구사항을 고려하여 적절한 패러다임과 언어를 선택하는 것이 중요합니다.
